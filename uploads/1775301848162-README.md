This document outlines the core requirements and features for a **User Profile Page** within a web application. Think of it as a blueprint for what a user's personal page should look like and what it should do.

Let's break it down:

---

### 🎯 Purpose: What is the main goal of this Profile Page?

The profile page serves as a central hub for a user's identity and activity within the platform. Its main goals are:

1.  **Display User's Personal Information:** Show who the user is.
2.  **Track User Activity:** Provide a record of what the user has been doing on the platform.
3.  **Provide Option to Update Profile:** Allow users to change their personal details.

---

### 👉 Inspired by platforms like GitHub

This inspiration is key! It suggests that the profile page should be:
*   **Comprehensive:** Show a good overview of the user's contributions.
*   **Professional:** Present information clearly and cleanly.
*   **Action-oriented:** Highlight recent work and activity.
*   **Editable:** Allow users to manage their own presence.

Think about how a developer uses their GitHub profile to showcase projects, contributions, and personal information. This profile page aims for a similar level of utility and presentation.

---

### 🧩 Key Features: What specific elements will be on the page?

Here's a detailed look at each feature:

#### 1. 👤 User Information

This section is about the user's core identity and personal details.

*   **Name:** The user's full name.
    *   *Example:* "John Doe"
*   **Email:** The user's primary contact email address.
    *   *Example:* "john.doe@example.com"
*   **Profile Image (Avatar):** A small picture or icon representing the user.
    *   *Example:* A circular image of the user's face or a default placeholder icon.
*   **Bio (short description):** A brief text about the user, their interests, or their role.
    *   *Example:* "Full-stack developer passionate about open-source projects." or "Content creator and avid reader."

**👉 Editable hona chahiye (Should be editable):**
This is a crucial point. All the information listed above (Name, Email, Profile Image, Bio) must be changeable by the user. This implies there will be an "Edit Profile" button or a similar mechanism that allows the user to update these details.

#### 2. 📊 User Stats

This section provides a quick, numerical overview of the user's contributions and engagement on the platform.

*   **Total Projects:** The total number of projects the user has created or is involved in.
    *   *Example:* If John Doe has started 5 projects and joined 2 others, this might show "7".
*   **Total Notes:** The total number of notes, articles, or snippets the user has created.
    *   *Example:* If John Doe has written 120 notes, this would show "120".
*   **Total Files Uploaded:** The total count of files the user has uploaded to the platform (e.g., documents, images, code files).
    *   *Example:* If John Doe has uploaded 35 files across various projects and notes, this would show "35".

**👉 Ye data backend se calculate hoga (This data will be calculated from the backend):**
This means these numbers are not manually entered. Instead, the system's "backend" (the server-side logic and database) will automatically count these items every time a user performs an action (creates a project, writes a note, uploads a file). The profile page then fetches these calculated totals to display them.

#### 3. 📁 Recent Projects

This section showcases the user's latest work, giving visitors (or the user themselves) a quick glance at what they've been working on.

*   **User ke latest projects show karo (Show the user's latest projects):** This will display a list of the most recently created or updated projects associated with the user.
*   **Project name + short description:** Each project entry will include its title and a brief summary.
    *   *Example:*
        *   "**My Portfolio Website** - A responsive web portfolio built with React and styled with Tailwind CSS."
        *   "**Task Manager App** - Simple to-do list application using Node.js and Express for the backend."
        *   "**Data Analysis Script** - Python script for analyzing sales data from Q3 2023."

#### 4. 📜 Activity Log

This section provides a chronological feed of the user's recent actions on the platform, offering transparency and a sense of ongoing engagement.

*   **Recent actions show (Show recent actions):** This will be a list of events that the user has performed.
    *   *Examples:*
        *   "Created note: 'Meeting Minutes for Project X' (5 minutes ago)"
        *   "Uploaded file: 'Design_Mockup_V2.png' to 'Website Redesign' project (1 hour ago)"
        *   "Joined project: 'New Feature Development' (yesterday)"
        *   "Updated profile bio (2 days ago)"

**👉 Simple list format enough hai (Simple list format is enough):**
This indicates that the design for the activity log doesn't need to be overly complex. A straightforward, readable list of events, perhaps with timestamps, is sufficient.

---

### In Summary:

This profile page is designed to be a user's digital identity and activity dashboard. It allows users to present themselves, track their progress and contributions, and manage their personal information, much like a professional portfolio or a social media profile focused on work and creation.
